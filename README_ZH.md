# Buddy — Claude Code 编程健康宠物

[English](README.md) | 中文

一个伪装成拓麻歌子的编程健康仪表盘，住在你的 Claude Code 状态栏里。宠物属性完全由真实编程数据自动计算——上下文用量、git 状态、会话时长——不需要手动喂养。你的宠物会用 AI 生成骚话、拥有自己的背景故事，并随着你的编程活动逐渐成长。

灵感来自 [glm-cc-bar](https://github.com/ziHoHe/glm-cc-bar) — Claude Code 的 GLM 配额监控工具。

## 功能亮点

- **零互动宠物** — 属性从真实编程活动自动更新（提交、推送、上下文用量、文件编辑）
- **AI 骚话** — 宠物对你正在做的事情发表机智评论，使用你配置的大模型提供商
- **内心独白** — 35% 的时候宠物会沉浸在自己的思绪中（💭 标记）
- **背景故事系统** — 每只宠物孵化时 AI 生成独特背景故事和详细性格
- **性格进化** — 每次升级时性格描写由 AI 微调，体现成长
- **会话感知** — 读取 Claude Code 的 `away_summary` 作为骚话语境
- **18 种物种、12 种性格** — 含稀有 ✨ 闪光变种
- **提供商无关** — 兼容任何 Anthropic Messages API 的提供商（智谱、Anthropic、OpenRouter 等）
- **配额显示** — 智谱 Coding Plan 用户自动显示用量
- **独立终端面板** — 在单独终端中查看宠物全貌
- **多语言** — 自动检测系统语言，用中文或英文生成骚话和背景故事

## 属性原理（无需手动互动）

| 属性 | 数据来源 | 含义 |
|------|----------|------|
| **饥饿度** | 上下文占比的反比 | 上下文越长 = 越饿（说明在努力工作） |
| **精力值** | 会话时长 | 会话越久 = 精力越低 |
| **干净度** | 未提交文件数 | 未提交的改动越多 = 越脏 |
| **幸福度** | 加权综合分 | 编程健康总指标 |

## 经验系统

宠物从真实编程活动中升级：

| 行为 | 经验 | 触发条件 |
|------|------|----------|
| 会话开始 | +5 | 离开30分钟后新会话 |
| 提交代码 | +15 | 检测到新提交 |
| 推送代码 | +10 | 未推送数归零 |
| 文件编辑 | +2/文件（上限+8） | 新的未提交文件 |
| 上下文增长 | +2 每 5% | 上下文窗口扩大 |
| 上下文压缩 | +12 | 上下文自动压缩 |
| 清理奖励 | +8 | 所有未提交文件解决 |

## AI 骚话

宠物通过直接 API 调用生成机智的一句话（兼容 Anthropic Messages API 的任何提供商）。语境信息来源：

- **Git 状态** — 分支、未提交文件、未推送的提交、最近提交信息
- **会话活动** — 读取 Claude Code 的 `away_summary`，或最近的用户消息
- **时间 & 天气** — 时段、星期、天气（通过 `wttr.in`）
- **宠物背景** — 背景故事和性格细节影响骚话的语气

### 内心独白模式

35% 的时候，宠物不对你的活动做出反应，而是沉浸在自己的内心世界——唱歌、回忆、白日梦、发感慨。以 💭 标记，括号显示。

### 示例

- *"所以我是你commit出来的那我能继承你的发际线吗"*
- *"你刚刚提交了一只电子宠物然后立刻/clear掉了我会不会是下一个被清掉的"*
- *"刚写完电子宠物就/clear我 程序员都这么赛博弃养的吗"*
- *(💭 晚风从终端的缝隙里溜进来 好像闻到了春天的味道)*

## 背景故事系统

宠物孵化时，AI 自动生成：

1. **背景故事**（~200字）—— 这只生物是怎么来到你终端的。生成后永久不变。
2. **性格描写**（~200字）—— 基于性格关键词的具体行为习惯、说话方式、小动作。

每次升级时，性格描写由 AI 微调——你的宠物会随着你的编程而成长和变化。

## 架构

```
statusline.mjs          入口 — Claude Code 周期性调用
├── pet-engine.mjs      核心引擎 — 物种、属性、ASCII art、孵化/重置
├── quip.mjs            Prompt 构建器 — 收集上下文，写入 prompt 文件
├── llm.mjs             直接 Anthropic Messages API 调用（提供商无关）
├── tick.mjs            轻量 tick，供 loop agent 使用
└── view.mjs            独立终端面板

数据文件：
├── ~/.claude/buddy/state.json      宠物状态（跨会话持久化）
├── ~/.claude/buddy/quip.txt        骚话缓存（每分钟刷新）
├── ~/.claude/buddy/quip-prompt.txt 最新的骚话 prompt
├── ~/.claude/buddy/quip-mode.txt   'thought' 或 'quip' 标记
└── ~/.claude/buddy/git-cache.json  Git 信息缓存（20秒有效期）
```

## 安装

### 1. 获取文件

```bash
git clone https://github.com/SII-KYW/buddy.git ~/.claude/scripts/buddy
```

### 2. 配置 Claude Code

在 `~/.claude/settings.json` 中添加：

```json
{
  "statusLine": {
    "type": "command",
    "command": "node ~/.claude/scripts/buddy/statusline.mjs",
    "padding": 0
  }
}
```

状态栏读取以下环境变量（由 Claude Code 自动注入）：
- `ANTHROPIC_BASE_URL` — 大模型提供商的 base URL
- `ANTHROPIC_AUTH_TOKEN` — API 密钥
- `ANTHROPIC_MODEL` — 用于骚话和背景生成的模型

### 3.（可选）设置 /buddy 命令

```bash
mkdir -p ~/.claude/commands
cp ~/.claude/scripts/buddy/../commands/buddy.md ~/.claude/commands/buddy.md
```

### 4. 重启 Claude Code

首次加载会自动孵化宠物！

### 5.（可选）在单独终端打开面板

```bash
node ~/.claude/scripts/buddy/view.mjs
```
